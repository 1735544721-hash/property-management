第2章 相关理论知识和技术介绍

2.1 Spring Boot框架介绍

Spring Boot 是基于 Spring 生态的快速开发框架，提供自动配置与 Starter 依赖管理机制，使开发者能够以“约定优于配置”的方式快速搭建可运行的应用。其内置的 Web 容器与完善的监控、配置体系，降低了传统 Java Web 项目的部署与维护成本，适合构建面向业务的轻量级服务。

在本系统中，Spring Boot 3.2 作为后端核心框架，统一承载 REST 接口、权限控制与业务逻辑。结合 Spring Security、JWT 与 Validation 等组件，实现登录鉴权、接口保护与参数校验；与 MyBatis-Plus 配合完成数据访问层封装，保证物业业务流程在服务端可维护、可扩展。

2.2 Vue.js前端框架

Vue.js 是渐进式 JavaScript 前端框架，采用响应式数据绑定与组件化开发模式，能够通过模板语法快速构建交互式界面。其生态中包含路由、状态管理与构建工具，配合现代化的构建链路，可实现模块化开发与高效迭代。

本系统前端基于 Vue 3 与 Vite 构建，结合 Element Plus 组件库实现住户端与管理端的统一界面风格。通过 Vue Router 组织页面导航、Pinia 管理全局状态、Axios 与后端 API 交互，实现报修、缴费、公告、投诉等核心业务的前端展示与操作。

2.3 MyBatis-Plus数据访问框架

MyBatis-Plus 是对 MyBatis 的增强框架，提供通用 CRUD、条件构造器、分页插件与代码生成等能力，在保留原有灵活性的基础上减少大量重复 SQL 编写。其 Lambda 查询与链式调用方式提升了数据访问的可读性与开发效率，适合业务模型较多的管理系统。

本系统在 DAO 层采用 MyBatis-Plus 统一数据访问规范，围绕楼栋、房屋、报修、费用、公告、投诉等核心表实现增删改查与分页查询，结合条件构造器完成多维检索需求，保证数据操作的规范性与扩展性。

2.4 MySQL数据库

MySQL 是常用的关系型数据库，支持 ACID 事务、索引优化与多种存储引擎，具备稳定可靠与易运维等特点。其完善的查询优化与事务机制能够满足业务系统对数据一致性与并发访问的需求。

本系统使用 MySQL 存储物业管理的基础数据与业务流水，包括住户信息、房屋档案、报修工单、费用账单与投诉记录等，通过规范化表结构与外键关联实现业务数据的可追溯与可统计。

2.5 Java语言

Java 是面向对象的编程语言，具有跨平台、生态成熟与安全性高等特点，适合构建中大型业务系统。依托 JVM 与丰富的开源框架，Java 在企业级应用开发中具备良好的性能与可维护性。

本系统后端以 Java 17 为开发语言，结合 Spring Boot 与 MyBatis-Plus 实现业务逻辑与数据访问层的分层设计。通过面向对象的领域建模与异常处理机制，确保物业管理业务流程稳定可控，并便于后续功能扩展。
