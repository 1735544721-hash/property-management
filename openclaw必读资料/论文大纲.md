4.2.2 逻辑设计

内容撰写参考【系统核心数据表围绕用户、业务、交互三类场景设计，不同类别数据表有明确架构与核心功能：用户类表（如创业用户表、学生表、管理员表）存储角色信息，业务类表（如课程信息表、项目信息表、融资信息表）承载核心服务数据，交互类表（如评论表、我的收藏表）记录用户操作。下文将按类别阐述各核心数据表的具体构建过程，确保数据模型贴合系统功能需求。

表 4-1 系统管理员表（admins）

| 编号 | 字段名     | 类型      | 长度 | 是否非空 | 是否主键 | 注释                    |
| ---- | ---------- | --------- | ---- | -------- | -------- | ----------------------- |
| 1    | id         | int       | 11   | 是       | 是       | 管理员 ID               |
| 2    | yonghuming | varchar   | 50   | 否       | 否       | 登录用户名              |
| 3    | pwd        | varchar   | 50   | 否       | 否       | 登录密码                |
| 4    | status     | varchar   | 10   | 否       | 否       | 账号状态（启用 / 禁用） |
| 5    | level      | varchar   | 10   | 否       | 否       | 权限等级                |
| 6    | addtime    | timestamp | -    | 是       | 否       | 创建时间                |

admins 表是平台最高权限账户的“总控台”，仅 6 个字段却完整记录登录身份、权限等级与生命周期。通过 status 字段可一键禁用危险账号，level 字段为后续 RBAC 细粒度授权预留扩展点；addtime 时间戳让超级账户的创建轨迹可审计，满足高校对后台人员“最小权限+操作留痕”的合规要求。

5.1 测试目的

内容参考【大学生校园创业系统聚焦校园创业场景，需满足用户注册、项目发布、融资申请、课程报名等核心需求，测试旨在全面验证系统功能完整性、数据准确性与运行稳定性。一方面，需确认各模块（如创业者管理、项目管理、融资对接、课程学习）协同无断层，确保用户操作流程顺畅，数据校验机制（如唯一性、格式合法性）有效拦截无效操作；另一方面，需模拟校园内多用户同时使用场景，排查高并发下系统响应与稳定性问题，保障系统上线后能可靠支撑学生与创业者的日常使用，符合校园创业服务的实际需求。】

5.3 测试结果分析

内容参考【本次测试围绕大学生校园创业系统的核心业务模块展开，重点覆盖用户注册、创业项目发布、融资信息申请、课程报名4大关键场景，共设计16条测试用例。

用户注册模块（表5-1）测试结果显示，系统对用户名唯一性校验、密码格式要求、必填项非空校验等机制均有效生效。测试用例01验证了新用户可正常完成注册流程并跳转至登录页面；用例02成功拦截了重复用户名的注册请求并给出明确提示；用例03和04则验证了系统对空用户名和空密码的非法输入进行了有效拦截，避免了无效数据的录入。

创业项目发布模块（表5-2）测试表明，项目信息的字段校验逻辑完善。测试用例01验证了完整信息的项目可成功发布并在列表中展示；用例02和03分别对项目编号唯一性和项目名称非空性进行了验证，系统均能准确识别并拦截；用例04对资金字段的数字格式校验有效，确保了项目资金数据的规范性。

融资信息申请模块（表5-3）测试验证了融资申请流程的完整性与数据合法性。测试用例01显示正常融资申请可顺利提交并生成记录；用例02和03分别对项目编号合法性和融资方非空性进行了校验，防止了关联无效项目或缺失关键信息的申请；用例04对融资金额的负数输入进行了有效拦截，保障了资金数据的合理性。

课程报名模块（表5-4）测试确认了课程学习功能的稳定性。测试用例01验证了已登录用户可成功报名未选课程；用例02的重复报名校验有效避免了同一用户的重复报名；用例03对不存在课程的非法请求进行了拦截；用例04则验证了未登录用户会被引导至登录页面，权限控制机制运行正常。

功能层面，各模块核心操作流程无断点，从用户注册时的信息校验，到项目发布、融资申请的字段合法性检查，再到课程报名的状态验证，各项数据校验机制均有效生效，能精准拦截无效操作，避免错误数据进入系统。数据层面，前后端数据同步及时，无论是用户账号信息、创业项目详情，还是融资申请记录、课程报名状态，均能准确存储与展示，无数据丢失、错乱或延迟问题。模拟校园内多用户同时在线操作的场景时，系统响应流畅，无卡顿、闪退情况；整体来看，系统功能与校园创业场景的用户需求高度契合，运行稳定可靠，完全具备上线条件。】
